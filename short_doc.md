# Clodex — краткий обзор продукта

## Что это

Clodex — локальная агентная IDE для разработки программного обеспечения.

Она объединяет:

- чат с AI-агентом;
- работу с файлами и Git;
- терминал;
- встроенный браузер;
- модели разных провайдеров;
- MCP, навыки и плагины;
- локальное, Docker, SSH и облачное выполнение;
- долгосрочную память задач;
- безопасные приложения и инструменты, создаваемые агентом.

Пользователь описывает результат, а агент анализирует проект, планирует работу,
изменяет код, запускает проверки и показывает готовый результат. Опасные
действия остаются под контролем пользователя и политик безопасности.

## Как выглядит работа

1. Пользователь открывает проект.
2. Создаёт задачу и описывает результат.
3. Агент анализирует структуру проекта и инструкции.
4. Выбирает модель.
5. Читает файлы, запускает команды и изменяет код.
6. Сохраняет важные факты и решения.
7. Запрашивает подтверждение для опасных действий.
8. Запускает тесты.
9. Показывает diff и итог.
10. Задачу можно продолжить после перезапуска или перенести в другую среду.

## Основные возможности

### AI-задачи

- создание, поиск, продолжение и архив;
- цели, прогресс и бюджеты;
- fork задачи с сохранением контекста;
- короткие статусы во время работы;
- отдельный финальный результат.

### Код

- чтение и изменение файлов;
- поиск по проекту;
- Git и worktrees;
- просмотр diff;
- принятие или отклонение изменений;
- review GitHub pull request;
- защищённый merge.

### Выполнение

- локальный терминал;
- Docker runner;
- SSH runner;
- собственный runner;
- cloud tasks;
- артефакты и receipts.

### Браузер

- вкладки и навигация;
- визуальный контекст;
- разрешения;
- контроль сетевых адресов;
- журнал разрешённых и заблокированных запросов.

### Модели

- Clodex Gateway;
- собственные API-ключи;
- OpenAI-compatible endpoints;
- локальные модели;
- выбор модели по задаче;
- учёт токенов, бюджета и лимитов.

### Память

- полная история задачи;
- важные факты и пользовательские ограничения;
- связь факта с источником;
- обнаружение устаревших сведений;
- обработка противоречий;
- краткие и долгосрочные summaries;
- восстановление после перезапуска.

### Расширения

- MCP;
- skills;
- plugins;
- private plugin sources;
- scheduled automations;
- приложения, создаваемые агентом.

## Безопасность

Clodex не полагается только на инструкции модели:

- модель не может самостоятельно выдать себе разрешение;
- команды и сеть проверяются отдельной policy;
- разрешение привязано к конкретному действию;
- повреждённый контекст блокируется;
- generated apps работают в sandbox;
- секреты не выдаются расширениям напрямую;
- телеметрия не должна содержать промпты, файлы, команды и ключи;
- удалённое выполнение связано со snapshot, lease и receipt.

## Чем продукт отличается

1. Это полноценная IDE, а не отдельный чат.
2. Контекст длительных задач сохраняется.
3. Поддерживаются Local, Docker, SSH и Cloud.
4. Можно выбирать провайдеров и модели.
5. Решения модели отделены от разрешений.
6. Платформа расширяется через MCP, plugins, skills и runners.
7. Новые возможности включаются только после проверок и наблюдений.

## Текущий статус

Архитектурное ядро реализовано:

- задачи и Agent Core;
- память;
- маршрутизация моделей;
- security policy;
- сетевой контроль;
- local и remote execution;
- перенос сессий;
- generated-app runtime;
- release-readiness infrastructure.

Продвинутые возможности остаются выключенными до реальных наблюдений и
подтверждения.

Перед production требуются:

- официальный защищённый repository;
- release signing;
- наблюдения на реальных установках;
- macOS, Windows и Linux acceptance;
- canary;
- monitoring и rollback.
