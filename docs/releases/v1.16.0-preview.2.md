# Clodex `v1.16.0-preview.2` release plan

This plan is committed release metadata. It prepares a technical preview but
does not dispatch a workflow, create a tag, or publish a GitHub Release.

## Immutable inputs

- source branch: `main`
- source base at plan creation: `1bb676f9b6b89d0b8d245a17497e74094cb198c0`
- version: `1.16.0-preview.2`
- tag: `v1.16.0-preview.2`
- build channel: `prerelease`
- rollback target: `v1.16.0-preview.1`
- manifest: `.release-notes/clodex-technical-preview.json`

The dispatch preflight resolves `main` to an immutable commit SHA and passes
that SHA to `_release-browser.yml`. It refuses an existing target tag, requires
the rollback tag, requires the exact changelog heading, and requires the
committed manifest to state that OAuth/WebAuth is not ready.

## Dispatch sequence

After the release candidate and external credentials are approved:

```bash
gh workflow run technical-preview-release.yml \
  --repo mereyabdenbekuly-ctrl/clodex-ide \
  --ref main \
  -f ref=main \
  -f manifest=.release-notes/clodex-technical-preview.json \
  -f confirm=RELEASE
```

The sequence is:

1. validate the committed preview plan and resolve the source SHA;
2. execute the reusable promotion gates;
3. create the idempotent `v1.16.0-preview.2` tag;
4. build, sign, notarize, smoke-test, and validate all platform artifacts;
5. publish a GitHub **pre-release** only when every matrix build succeeds.

`Prepare Release` remains the path for `alpha`, `beta`, and stable package
versions. It is intentionally not used for the public `vX.Y.Z-preview.N` tag
series because it changes package metadata and emits `clodex@...` tags. After
merging a normal release PR, a maintainer must explicitly dispatch
`Auto Release`; merge alone does not publish artifacts.

## Artifact contract

GitHub Actions bundles:

- `clodex-macos-arm64`
- `clodex-macos-x64`
- `clodex-windows-x64`
- `clodex-linux-x64`

Expected macOS files:

- `clodex-prerelease-1.16.0-preview.2-arm64.dmg`
- `clodex-prerelease-darwin-arm64-1.16.0-preview.2.zip`
- `clodex-prerelease-1.16.0-preview.2-x64.dmg`
- `clodex-prerelease-darwin-x64-1.16.0-preview.2.zip`

Expected Windows files:

- `clodex-prerelease-1.16.0-preview.2-x64-setup.exe`
- `clodex-prerelease-1.16.0-preview.2-x64-full.nupkg`
- `RELEASES-win32-x64`

Linux `.deb` and `.rpm` names are generated by Electron Forge and validated by
package metadata rather than assumed by filename.

Every platform emits a schema-version `1` JSON validation manifest and a
SHA-256 file:

- `macos-{arm64,x64}-1.16.0-preview.2.{json,sha256}`
- `windows-x64-1.16.0-preview.2.{json,sha256}`
- `linux-x64-1.16.0-preview.2.{json,sha256}`

## Authentication scope

Existing provider credential configuration is the supported authentication
path. OAuth/WebAuth is not included in this preview and must not be represented
as available in release notes or product messaging.

## Rollback

If acceptance or canary monitoring fails:

1. stop expansion of the prerelease update feed;
2. pin the prerelease feed back to `v1.16.0-preview.1`;
3. preserve the failed manifests and checksums for diagnosis;
4. do not reuse or move the `v1.16.0-preview.2` tag.

Changing the update feed requires the configured update-server operator and is
an external operational action, not part of this workflow.
