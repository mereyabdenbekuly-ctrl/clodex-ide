# Privacy Policy

**Effective date: July 21, 2026**

This policy explains how the Clodex project handles information when you use the project website or the Clodex Agentic IDE.

## Website data

The website may process standard server logs such as IP address, request time, browser type, requested URL, and error information. These records are used to operate, secure, and troubleshoot the service.

Optional PostHog analytics are off by default. The website does not initialize the PostHog browser client or send PostHog requests unless you select **Allow analytics**. Selecting **Keep analytics off** leaves the client disabled.

When enabled, website analytics are limited to sanitized page-view events and a small allowlist of coarse browser and SDK metadata. URLs are reduced to their origin and path; query strings and fragments are never included. Autocapture, session recording, person profiles, device-model collection, surveys, product tours, conversations, performance capture, and external PostHog script loading are disabled.

The website stores your analytics choice in a first-party preference cookie for up to one year. Use **Privacy choices** in the website footer to change or withdraw that choice. Withdrawal closes the local capture gate before the PostHog client is opted out, so subsequent page navigation is not sent to PostHog.

## Accounts and authentication

If you choose to sign in, the authentication service may process the account identifier you provide, such as an email address or Telegram account identifier, together with session and security metadata needed to authenticate you.

## Desktop application and project data

Clodex is designed to keep project data local unless you choose a feature that requires transmission. Data may leave your machine when you use a model provider, remote runner, cloud task, plugin, MCP server, sharing feature, authentication service, or another external integration.

The selected service receives the information required to perform the requested action. Its own privacy policy and retention rules apply. Review provider settings before sending source code, credentials, personal data, or confidential material.

Credentials should be stored through the operating system's protected credential facilities where supported. Do not place secrets in prompts, repositories, screenshots, logs, or issue reports.

## Telemetry

Optional product telemetry may include feature usage, diagnostics, application version, platform information, and error metadata. It is intended to exclude source files, prompts, responses, credentials, and terminal output unless a separate diagnostic flow explicitly asks you to include particular material and obtains the relevant choice. Standard IDE analytics do not send email addresses or the raw persistent machine identifier. Community Observed analytics use the separately disclosed, domain-separated pseudonymous installation identifier only after the explicit anonymous-statistics choice.

## Sharing and sale of data

The project does not sell personal data. Information may be processed by infrastructure and integration providers only as needed to provide the selected functionality, protect the service, or comply with applicable law.

## Retention and deletion

Local application data remains on your device until you remove it or use an in-product deletion control. External services keep data according to their own policies. Operational logs are retained only as long as reasonably necessary for security, reliability, and abuse prevention.

## Security

Clodex uses technical safeguards appropriate to a developer tool, but no system can guarantee absolute security. Keep the application updated, review permissions, rotate exposed credentials immediately, and report suspected vulnerabilities privately.

## Contact

For privacy requests, contact **support@clodex.xyz**. Report suspected
vulnerabilities through [GitHub private vulnerability reporting](https://github.com/mereyabdenbekuly-ctrl/clodex-ide/security/advisories/new),
not by public issue or ordinary email.

This policy may be updated as the project and its optional services evolve. Material changes will be reflected on this page with a new effective date.
